RAUD-22 Retrieval Cold-Start Remediation
Date: 2026-03-12
Prompt ID: RAUD-22
Finding IDs: M5, N-34

Prior status
- Not Fixed. The default resource retrieval path still had a cold-start full-load
  branch:
  `findByTypeSearchApi()` -> sparse lexical results ->
  `TopicResolver::resolveFromText()` -> `findByTopic()` ->
  `getAllResources()` -> `indexResources()` -> full `loadMultiple()`.
- Resource legacy fallback also depended on full corpus materialization through
  `findByTypeLegacy()` -> `getAllResources()`.
- FAQ legacy search still cold-started through
  `searchLegacy()` -> `getAllFaqsLegacy()` -> `loadByProperties()`.

- Fixed

Residual risks / unknowns
- `FaqIndex::getCategoriesLegacy()` still uses `getAllFaqsLegacy()` as a
  browse-only fallback when the FAQ lexical index is unavailable. That path is
  intentionally documented as residual and is outside the default retrieval
  proof surface.
- `ResourceFinder::getAllResources()` remains available for compatibility, but
  the normal request/search paths covered by `RAUD-22` no longer depend on it.
- No hosted cold-cache latency probe was run in Pantheon as part of this repo
  remediation; the proof here is structural and executable at the code/test
  level.

Rollback notes
- Roll back the `ResourceFinder` and `FaqIndex` bounded candidate loaders, the
  cold-start guard test, and the related audit documentation as one code/docs
  bundle so retrieval behavior and evidence stay aligned.
- No schema migration or content rewrite was introduced by this remediation.
- Highest-risk rollback surfaces are the request-path retrieval helpers in
  `ResourceFinder` and `FaqIndex`, because reverting only one side could
  restore hidden full-load behavior without restoring the original tests.

Still-unverified surfaces
- Pantheon cold-cache retrieval behavior after deployment
- Browse-only FAQ category fallback behavior when the lexical FAQ index is
  unavailable
- Environment-specific memory/latency impact under real content volume outside
  the local executable test suite
