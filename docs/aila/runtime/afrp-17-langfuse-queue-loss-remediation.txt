  with a clean local queued stale-item drill proving `L2:QueueDrain` on the
  worker path.

  non-actionable. Queue health/SLO alerting only reacted to queue depth/age,
  `send_partial_207` only preserved rejected-event counts on the most recent

## Findings

### Prior Langfuse queue-loss posture

- `drop_max_depth`: counted in state and logged, but not classified or alerted.
- `discard_invalid_shape`: logged and batch-counted only.
- `discard_missing_enqueued_at`: logged and batch-counted only.
- `discard_stale`: logged and batch-counted only.
- `discard_disabled`: logged and batch-counted only.
- `discard_missing_credentials`: logged and batch-counted only.
- `discard_non_retryable_http`: logged/error-counted only.
- `send_partial_207`: counted only as a batch outcome; aggregate rejected-event
  totals were not retained.
- `retryable_suspend`: counted only as a batch outcome; visible only if an
  operator inspected status/probe output or queue depth/age later degraded.
- terminate-side enqueue/payload exception: warning-log only; no queue-loss
  counter or operator summary.

### Post-change posture

- Queue-loss outcomes are now classified as `alertable_loss`,
  `informational_loss`, `retry_only`, or `success`.
- Aggregate per-outcome totals now retain queue-item/event counts plus
  `lost_queue_items` / `lost_event_count`.
- `drop_enqueue_failure` now captures terminal enqueue failures from
  `LangfuseTerminateSubscriber`.
- `send_partial_207` now counts rejected events as partial loss and becomes
  alertable only when `error_count > 0`.
- `SloAlertService` now emits cooldowned warnings for new alertable queue-loss
  outcomes without making historical queue-loss counters turn `/health` sticky
  red.
- `/assistant/api/health` and `/assistant/api/metrics` now expose the same
  summary under `queue.export`.
- Admin/report output now exposes action-required state plus alertable and
  informational loss totals.

### Loss-path accounting matrix

| Loss path | Prior posture | Post-change posture | Final policy |
|---|---|---|---|
| `drop_max_depth` | Counter + log only | Structured summary + queue-loss alert | Alertable |
| `drop_enqueue_failure` | Log only | Structured summary + queue-loss alert | Alertable |
| `discard_invalid_shape` | Log + raw counter only | Structured summary + queue-loss alert | Alertable |
| `discard_missing_enqueued_at` | Log + raw counter only | Structured summary + queue-loss alert | Alertable |
| `discard_stale` | Log + raw counter only | Structured summary + queue-loss alert | Alertable |
| `discard_disabled` | Log + raw counter only | Structured summary only | Informational |
| `discard_missing_credentials` | Log + raw counter only | Structured summary + queue-loss alert | Alertable |
| `discard_non_retryable_http` | Log + raw counter only | Structured summary + queue-loss alert | Alertable |
| `send_partial_207` rejected events | Last batch metadata only | Aggregate rejected-event totals + queue-loss alert | Alertable when `error_count > 0` |
| `retryable_suspend` | Log + raw counter only | Structured summary only | Informational retry |
