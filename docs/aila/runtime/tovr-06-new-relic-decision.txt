TOVR-06 New Relic Binary Decision — RETIRE
===========================================
Date: 2026-03-16
Scope: AILA-owned New Relic integration (not Pantheon platform-level APM)

1) Prior status
---------------
- Browser snippet injection: runtime-secret path exists in settings.php,
  b5subtheme.theme, html.html.twig, ilas_site_assistant.module, and
  observability.js. Sampled local/dev/test/live pages all rendered
  newRelic.browserEnabled=false. No browser snippet value was ever active.
- Deploy change tracking: pantheon.yml referenced
  scripts/quicksilver/new-relic-change-tracking.php. Recent test/live deploy
  logs reported "not a valid path." terminus new-relic:info returned empty
  fields.
- Browser action/error hooks: observability.js contained configureNewRelic(),
  newrelic.noticeError(), and newrelic.addPageAction() calls, all guarded by
  window.newrelic which was never present (no snippet loaded).
- DDEV local scaffold: config.newrelic.yaml.example and
  Dockerfile.newrelic.example existed as opt-in example files, never activated.
- Pantheon secret presence: NEW_RELIC_BROWSER_SNIPPET, NEW_RELIC_API_KEY,
  NEW_RELIC_ENTITY_GUID_APM, and NEW_RELIC_ENTITY_GUID_BROWSER names were
  present on dev/test/live but values produced no observable effect.

2) Binary decision: RETIRE
--------------------------
Rationale:
  a) Zero data flowing — dormant on all environments.
  b) Quicksilver change tracking broken — Pantheon logs: "not a valid path."
  c) terminus new-relic:info returns empty fields — no entity-side proof.
  d) Browser error capture duplicates Sentry (proven, operational, with replay
     + PII scrubbing).
  e) PRODUCTION_AUDIT ranked tools by ROI — New Relic absent from
     recommendations.
  f) RAUD-28 audit: "Remove New Relic scaffold or document as intentionally
     disabled."
  g) ilas:assistant:action events route only to NR with no dashboard/alert
     consumers.
  h) Sentry has nonprofit free tier; NR cost path unclear.
  i) CWV/RUM gap is real but served by GA4 or revisitable as a platform-level
     (not AILA) concern.

3) Verification level: full code cleanup with atomic test updates

All AILA-owned New Relic wiring removed in a single commit. Platform-level
Pantheon New Relic APM is a separate concern and is not touched.
