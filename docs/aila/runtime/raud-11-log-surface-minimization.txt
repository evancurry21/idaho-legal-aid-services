## Residual risks / unknowns

- Historical third-party telemetry already exported before deployment cannot be
  retroactively scrubbed by repo changes; `update_10007` only prevents queued
  pre-RAUD-11 batches from exporting after deploy.
- The SQLite override proves kernel behavior in this workspace, but production
  rollout still needs the real database update hook to execute successfully in
  deploy pipelines.
- Widget hardening smoke coverage was updated, but the browser-only JS smoke
  file was not executed in a DOM harness during this remediation run.

## Rollback notes

- Code rollback: revert the PHP/JS/doc changes listed above.
- Database rollback: if `update_10007` has already run, restoring the old
  text-bearing columns would require a custom schema/data rollback because the
  migration intentionally drops `sanitized_query` and `redacted_message`.
- Operational rollback: if Langfuse export behavior must be reverted quickly,
  disable Langfuse before restoring older payload contracts to avoid exporting
  mixed-schema batches.
