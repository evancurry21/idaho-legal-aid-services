AFRP-09 Langfuse Operator Truth Remediation
Date: 2026-03-24
Verification level tag: repo remediated / local verified / hosted deployment pending

Prior Langfuse runtime-truth posture
- AFRP-02 (2026-03-18) established general stored-versus-effective runtime truth
  tooling (`ilas:runtime-truth`, `ilas:langfuse-status`, admin report) and documented
  the secret-gated override pattern. However, Langfuse-specific operator experience
  still had gaps:
  - `ilas:langfuse-probe` guard messages told operators to "Set langfuse.enabled=true
    in ilas_site_assistant.settings" when the probe failed, which was misleading because
    enablement depends on secret presence in `settings.php`, not config edits.
  - Operators who only ran `ilas:langfuse-probe` and saw it fail had no guidance to
    run `ilas:langfuse-status` for the full stored-vs-effective picture.
  - `LangfuseTracer::isEnabled()` silently returned FALSE when credentials were absent,
    with no log entry explaining why tracing was off for a given request.
  - There was no single-command diagnostic to quickly assess Langfuse readiness without
    actually sending a probe.

Post-change posture
- Probe guard messages now reference LANGFUSE_PUBLIC_KEY/LANGFUSE_SECRET_KEY secrets
  and suggest `drush ilas:langfuse-status` for override channel inspection. No message
  tells operators to edit config directly.
- New `--diagnose` option on `ilas:langfuse-probe` prints a focused readiness JSON
  report with: verdict (READY, DISABLED_NO_SECRETS, ENABLED_NO_CREDENTIALS), stored
  and effective enabled state, credential presence booleans, environment label, sample
  rate, queue health, export outcomes, and an operator-facing suggestion when not ready.
  No probe is sent, no secrets are leaked.
- `LangfuseTracer::isEnabled()` now logs a single entry per request when the enabled
  state resolves: `info` for config_disabled and credentials_absent (operator-actionable),
  `debug` for sampled_out and active (expected probabilistic behavior).
- Probe commands are now wired with `RuntimeTruthSnapshotBuilder`, `QueueHealthMonitor`,
  and `SloDefinitions` for richer diagnostic context.
- Runbook PHARD-02 section now includes `--diagnose` as the recommended first triage
  step before probe attempts.

Operator truth-path summary
- Quick triage: `drush ilas:langfuse-probe --diagnose`
  Outputs: verdict, stored/effective config, credential presence, environment,
  sample rate, queue health, export outcomes, actionable suggestion.
- Full status: `drush ilas:langfuse-status`
  Outputs: stored config, effective runtime, divergences, override channels,
  queue health, export outcomes.
- Connectivity test: `drush ilas:langfuse-probe --direct`
  Outputs: HTTP status, trace ID, ingestion success/error counts.
- Account-side proof: `drush ilas:langfuse-lookup <trace_id>`
  Outputs: sanitized account-side trace metadata.
