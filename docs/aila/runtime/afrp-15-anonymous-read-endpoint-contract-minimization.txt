AFRP-15 — Anonymous Read Endpoint Contract Minimization
=======================================================
Date: 2026-03-25
Prerequisite: AFRP-04

1. Prior Anonymous Read Contract
--------------------------------

Supported compatibility scope for this remediation:

  - Repo-proven callers/tooling only
  - Published README examples
  - No compatibility guarantee for undocumented anonymous fields

Anonymous API read surfaces in scope:

  a) GET /assistant/api/session/bootstrap
     - 200 body: raw CSRF token text
     - 200 headers: text/plain; charset=UTF-8, Cache-Control: no-store, private,
       X-Content-Type-Options: nosniff
     - 429 body: plain-text throttle message
     - 429 headers: Retry-After, text/plain; charset=UTF-8,
       Cache-Control: no-store, private, X-Content-Type-Options: nosniff
     - Caller proof:
       - assistant-widget.js fetchCsrfToken() consumes response.text()
       - functional bootstrap tests assert token + cookie continuity

  b) GET /assistant/api/suggest
     - 200 body: {"suggestions":[{"type","label","id"}]}
     - Additional 200-path metadata existed in one degraded-success case:
       reason_code=request_id on FAQ dependency loss
     - 429 body: {"suggestions":[],"error", "type":"rate_limit","request_id"}
     - 429 header: Retry-After
     - Caller proof:
       - README example documents only type/label/id
       - no shipped frontend code calls /suggest directly
       - runbook/runtime probes treat it as public anonymous JSON

  c) GET /assistant/api/faq
     - Query mode 200 body previously returned:
       results[] = {id, question, answer, url, source}
       count
     - ID mode 200 body returned:
       faq = {id, question, answer, url}
     - Category mode 200 body returned whatever FaqIndex::getCategories()
       emitted; runtime sample was {name, count}, but the controller did not
       explicitly lock that DTO
     - 404 ID mode body: {"error":"FAQ not found"}
     - 429 body: endpoint-shaped empty payload + error/type/request_id
     - 503 unavailable body: endpoint-shaped empty payload +
       error/type/error_code/request_id
     - Caller proof:
       - README example documents only id/question/answer/url for search mode
       - no shipped frontend code calls /faq directly
       - runbook/runtime probes treat it as public anonymous JSON

5. Residual Risks / Unknowns
----------------------------

  - The shipped widget still does not consume /suggest or /faq directly, so the
    public contract remains doc-and-tooling governed rather than browser-UI
    governed.
  - Direct curl captures in this run focused on success shapes. Throttle and
    unavailable shapes remain covered by unit/functional tests rather than
    fresh raw HTTP captures.
  - BrowserTestBase normalizes GET Cache-Control differently
    (must-revalidate/no-cache/private) than the direct DDEV runtime
    (no-store/private), so the functional assertion is intentionally written
    against the stable security intent rather than one exact cache string.
