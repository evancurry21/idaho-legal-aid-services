PRIOR PROVENANCE-ENFORCEMENT POSTURE

Signal                    Level      Behavior
─────────────────────     ─────────  ─────────────────────────────────────────
stale_source (per-item)   ADVISORY   Flag in governance_flags; no response change
unknown_freshness (item)  ADVISORY   Flag in governance_flags; no response change

Unknown freshness (agg)   NONE       Not counted as non-fresh; no enforcement
Partial staleness         NONE       No enforcement at all
enforcement_mode config   DECORATIVE Read from config, never consumed to gate behavior

Gaps:
  - Per-item freshness flags were purely advisory
  - All-unknown-freshness citations had ZERO enforcement
  - Partial staleness (some stale, some fresh) had ZERO enforcement
  - Stale citations caused only a text caveat, never a confidence downgrade
  - No structured freshness_profile visible in the response

ENFORCEABLE POLICY MATRIX
────────────────────────────────────────────────

Citation-required types:
  faq, resources, topic, eligibility, services_overview, search_results

Freshness classification of citations:
  fresh   — updated_at within max_age_days (180 default)
  stale   — updated_at exceeds max_age_days
  unknown — no updated_at resolvable (treated as non-fresh)

Enforcement rules for citation-required responses with sources:

  Condition              Enforcement                    Confidence Cap
  ────────────────────   ────────────────────────────   ──────────────
  All fresh              None                           Unchanged
  Partial non-fresh      partial_non_fresh enforcement  max(0.5, 1.0 - ratio*0.5)
  All non-fresh          all_non_fresh enforcement      0.5
  No sources             grounding_weak (existing)      0.3 + refusal at ≤0.5

  "Non-fresh" = stale OR unknown (precautionary principle)

Non-citation-required types (navigation, emergency, escalation, etc.):
  No freshness enforcement applied. ADVISORY flags only.

RESIDUAL RISKS / UNKNOWNS
────────────────────────────────────────────────

1. 3-source limit: addCitations() caps at 3 sources. If ordering places
   all stale sources first, freshness_profile reflects only those 3.
   Documented as known limitation.

2. Check 3 / Check 4 overlap: Both fire for all-stale-citations.
   Idempotency guard prevents duplicate caveats. Check 3 can be merged
   into Check 4 in a future cleanup AFRP.

3. Confidence multiplier aggressiveness: The 0.5 multiplier in the
   proportional formula (max(0.5, 1.0 - ratio*0.5)) may be too gentle
   for partial staleness. Tunable in future without API change.

4. Advisory-only paths remaining:
   - Per-item stale_source and unknown_freshness remain ADVISORY
     (they annotate items, not responses)
   - health_degraded snapshot status remains ADVISORY
   - These are intentional: per-item flags are informational,
     enforcement operates at response level

────────────────────────────────────────────────
ROLLBACK NOTES
────────────────────────────────────────────────

Set source_governance.freshness_enforcement.enabled: false in config
to disable all freshness enforcement. Effect:
  - enforceFreshnessPolicy() becomes no-op
  - No freshness_profile added to responses
  - No _freshness_enforcement or _freshness_confidence_cap flags set
  - Existing Check 3 (_all_citations_stale caveat) continues independently
  - Per-item ADVISORY annotations unaffected
