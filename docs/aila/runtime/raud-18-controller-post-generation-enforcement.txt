# RAUD-18 Controller-Owned Post-Generation Enforcement

- Prior status: Not Fixed
- Post-change status: Partially Fixed
- Verification level tag: VC-UNIT + VC-PURE + VC-QUALITY-GATE

## Evidence

### Pre-change controller-path gap

Code inventory before editing showed the controller consumed `_requires_review`
too narrowly:

- `web/modules/custom/ilas_site_assistant/src/Service/ResponseGrounder.php`
  set `_requires_review = TRUE` from unsafe `response['message']` content.
- `web/modules/custom/ilas_site_assistant/src/Controller/AssistantApiController.php`
  consumed that flag by replacing only `llm_summary`, leaving the public
  `message` field unchanged on the real response path.
- `web/modules/custom/ilas_site_assistant/tests/src/Unit/PostGenerationEnforcementTest.php`
  mirrored regex lists and `unset()` behavior instead of executing
  `AssistantApiController::message()`.

That meant controller-path enforcement could be removed or narrowed without the
existing tests failing, and a grounded unsafe `message` could still survive to
the final JSON response.

### Post-change controller ownership

The controller now owns the final block/replace decision:

- `web/modules/custom/ilas_site_assistant/src/Controller/AssistantApiController.php`
  now replaces the final public `message` when `_requires_review` is present,
  replaces `llm_summary` when present, strips internal enforcement flags before
  serialization, and returns enforcement metadata separately for tracing rather
  than leaking sentinel fields through the response payload.
- `web/modules/custom/ilas_site_assistant/src/Service/PostGenerationLegalAdviceDetector.php`
  is the shared deterministic detector used by both the controller and
  `LlmEnhancer`, removing duplicated post-generation pattern lists.
- `web/modules/custom/ilas_site_assistant/src/Service/LlmEnhancer.php`
  still performs defense-in-depth filtering, but the controller remains the
  authoritative final-response enforcer.
- `web/modules/custom/ilas_site_assistant/tests/src/Unit/PostGenerationEnforcementTest.php`
  now drives the real `message()` path with a real `ResponseGrounder`, stubbed
  `LlmEnhancer`, and assertions against the final JSON response contract.

## Residual risks / unknowns

- This remains `Partially Fixed` because post-generation detection is still
  heuristic pattern matching, even though controller enforcement is now real.
- Live LLM-generated output with `llm.enabled=true` was not exercised here; the
  controller-path proof used deterministic test doubles around the actual
  response-building path.
- Additional paraphrase families may still evade the shared detector until
  future corpus expansions add them.

## Rollback notes

- Roll back the controller enforcement changes, shared detector, controller-path
  test rewrite, and this artifact as one code-only bundle.
- No schema, config-shape, or database rollback is required.
