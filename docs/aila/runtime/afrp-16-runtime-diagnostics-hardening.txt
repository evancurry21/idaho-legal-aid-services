AFRP-16 Runtime Diagnostics Hardening
======================================
Date: 2026-03-25
Verification level: L0:Unverified (config inspection only, no probes sent)

Prior diagnostics posture
-------------------------
Operators needed to run multiple commands to assess assistant environment health:
  - `ilas:runtime-truth` for stored-vs-effective config divergences
  - `ilas:langfuse-status` for Langfuse queue and config health
  - `search-api:server-list`, `search-api:list`, `search-api:status` for index inventory
  - `ilas:sentry-probe` for Sentry connectivity
  - `ilas:langfuse-probe --direct` for Langfuse ingestion
  - Manual correlation of multiple JSON outputs to determine overall health

Missing indexes, disabled credentials, and override-driven toggles required combining
scattered data from different command outputs. No single artifact answered "is this
environment healthy, and what proof do I have?"

Post-change diagnostics posture
-------------------------------
Single command: `ilas:runtime-diagnostics`
Output: unified jq-parseable JSON artifact containing:
  - schema_version (semver, currently 1.0.0)
  - timestamp (ISO-8601)
  - environment (effective environment, Pantheon env, classification)
  - runtime_truth_summary (divergence count and field list)
  - diagnostics_matrix (~25 rows, each with fact_key, category, current_value,
    source, proof_level, proof_level_label, static_proof_ceiling,
    verification_command, assertion)
  - integration_status (sentry, langfuse, pinecone, voyage with proof levels)
  - credential_inventory (boolean-only, 10 credentials)
  - retrieval_inventory (index/server health from RetrievalConfigurationService)
  - degraded_mode_state (overall status, active degradations, feature gates off,
    missing credentials, failed facts)
  - verification_commands (VC-* aliases with command strings)

Exit code: 0 = no fail assertions, 1 = any fail assertion (enables CI/script gating)

Assertion semantics:
  - pass: fact is in healthy/expected state
  - fail: required fact is broken (e.g., required index missing)
  - degraded: non-ideal but allowed-degraded-mode applies
  - skipped: feature gate off, check does not apply

Residual risks/unknowns
------------------------
1. Diagnostics token not provisioned (diagnostics_token_present=false everywhere).
   Machine-auth HTTP path remains operationally unproven. Drush-based diagnostics
   work without the token.
2. Proof levels for all diagnostic facts default to L0:Unverified because the
   runtime-diagnostics command reads config only. Raising proof levels requires
   running probes (ilas:sentry-probe, ilas:langfuse-probe --direct, etc.) and
   is outside the scope of this command.
3. Pinecone live readiness still blocked (gate + acceptance + timeout separation).
4. Sentry browser source maps remain unverified (L1 only).
5. Cron/queue sustained-load behavior unobserved (B-04, outside audit scope).
