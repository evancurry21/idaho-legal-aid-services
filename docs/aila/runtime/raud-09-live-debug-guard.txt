# RAUD-09 Live Debug Metadata Guard Remediation Report

- Date: 2026-03-10
- Prompt ID: `RAUD-09`
- Finding IDs: `H3`, `N-25`
- Prior status: Open / partially verified. Response debug metadata was controlled only by `AssistantApiController::isDebugMode()` via `getenv('ILAS_CHATBOT_DEBUG') === '1'`, so a live environment with that variable set could expose routing/debug internals to any client.
- Post-change status: Partially Fixed
- Verification level tag: `Partially Fixed`

## Evidence

- Pre-edit enablement inventory:
  - Command summary: `rg -n "ILAS_CHATBOT_DEBUG|_debug|isDebugMode|debug metadata" web/modules/custom/ilas_site_assistant/src/Controller/AssistantApiController.php web/sites/default/settings.php web/sites/default/settings.pantheon.php web/modules/custom/ilas_site_assistant/src/EventSubscriber/SentryOptionsSubscriber.php scripts/chatbot-eval/README.md`
  - Result: the only response-debug enablement path was `AssistantApiController::isDebugMode()` reading `ILAS_CHATBOT_DEBUG`; every `_debug` attachment site in the controller depended on that boolean; `settings.php` and `SentryOptionsSubscriber.php` did not provide any live deny path.
- Post-edit repo guard check:
  - Command summary: `rg -n "EnvironmentDetector|ilas_site_assistant_debug_metadata_force_disable|isDebugMode|isLiveEnvironment" web/modules/custom/ilas_site_assistant/src web/modules/custom/ilas_site_assistant/ilas_site_assistant.services.yml web/sites/default/settings.php`
  - Result: live detection is now centralized in `EnvironmentDetector`; `settings.php` adds a live-only `ilas_site_assistant_debug_metadata_force_disable` flag; `AssistantApiController::isDebugMode()` now fails closed on the settings flag and on `live`.
