TOVR-08 Override-Aware Runtime Truth Verification
=================================================

Prompt ID: TOVR-08
Roadmap item: Tooling + Pinecone remediation -> Operations verification #1
Verification level tag: repo-remediated / local-post-edit-verified / pantheon-deployment-pending
Created: 2026-03-17
Last updated: 2026-03-17

1. Prior status
---------------

Before TOVR-08, the canonical `VC-RUNTIME-*` examples used direct `php:eval`
snapshots and older `config:get` habits were still easy to misuse as runtime
truth. Fresh 2026-03-17 rechecks reproduced the audited divergence:

- Stored config:
  - `config/ilas_site_assistant.settings.yml` keeps `langfuse.enabled=false`.
  - `config/raven.settings.yml` is absent.
- Fresh baseline commands:
  - Local:
    - `ddev drush config:get ilas_site_assistant.settings langfuse.enabled -y`
      -> `'ilas_site_assistant.settings:langfuse.enabled': false`
    - `ddev drush php:eval '$c=Drupal::config("ilas_site_assistant.settings"); echo ($c->get("langfuse.enabled") ? "true" : "false") . PHP_EOL;'`
      -> `true`
  - Pantheon `dev` / `test` / `live`:
    - `terminus remote:drush ... -- config:get ilas_site_assistant.settings langfuse.enabled -y`
      -> `false`
    - `terminus remote:drush ... -- php:eval '...langfuse.enabled...'`
      -> `true`
- Misleading prior habits:
  - `config:get raven.settings`
  - `config:get langfuse.settings`
  - `config:get ilas_site_assistant.settings langfuse.enabled`

3. Before / after procedure
---------------------------

Before:

- `config:get` or one-off `php:eval` snippets were used to infer runtime truth.
- Browser-only runtime was sampled separately with `/assistant`, but there was
  no single canonical sanitized helper for stored-versus-effective comparison.

After:

- Canonical `VC-RUNTIME-LOCAL-SAFE`:
  - `ddev drush status --fields=uri,drupal-version,db-status`
  - `ddev drush ilas:runtime-truth`
  - `curl -skL https://ilas-pantheon.ddev.site/assistant | rg ...`
- Canonical `VC-RUNTIME-PANTHEON-SAFE` after deploy:
  - `terminus remote:drush ... -- status --fields=uri,drupal-version,db-status`
  - `terminus remote:drush ... -- ilas:runtime-truth`
  - `curl "$BASE_URL/assistant" | rg ...`
