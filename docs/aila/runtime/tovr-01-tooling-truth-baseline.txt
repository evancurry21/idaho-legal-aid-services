## Direct code inspection summary

- `.github/workflows/quality-gate.yml` wires a repo-owned `Quality Gate` workflow for `push`, `pull_request`, and `workflow_dispatch`, with `PHPUnit Quality Gate` and `Promptfoo Gate` jobs.
- Protected-branch pushes in that workflow require `ILAS_ASSISTANT_URL` and run `scripts/ci/run-promptfoo-gate.sh --skip-eval --simulate-pass-rate 100`, which proves config-parity mode but not a guaranteed deploy-bound live eval.
- `.github/workflows/observability-release.yml` is manual-only (`workflow_dispatch`) and runs `scripts/observability/sentry-release.sh`.
- `scripts/observability/sentry-release.sh` prepares a Sentry release and uploads theme sourcemaps when auth, org/project settings, a release name, and `.map` files are available.
- `web/sites/default/settings.php` injects runtime Langfuse keys, Sentry DSNs, Pinecone key wiring, GA4 live-only settings, and the runtime-secret-gated New Relic browser snippet.
- `web/modules/custom/ilas_site_assistant/ilas_site_assistant.module` attaches browser observability settings only when Sentry browser mode or New Relic browser mode is enabled.
- `web/modules/custom/ilas_site_assistant/js/observability.js` emits browser errors/actions to Sentry and New Relic when those providers are enabled in `drupalSettings`.
- `web/themes/custom/b5subtheme/templates/page/html.html.twig` prints the New Relic browser snippet only when a runtime value is provided.
- `pantheon.yml` wires the Quicksilver hook `scripts/quicksilver/new-relic-change-tracking.php`.
- `.ddev/config.newrelic.yaml.example` and `.ddev/newrelic/Dockerfile.newrelic.example` are opt-in examples, not active local overlays.
