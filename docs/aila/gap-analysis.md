## Purpose and source of truth
This document provides a gap analysis and pragmatic improvement plan for ILAS Site Assistant (Aila) using the completed current-state audit as the authoritative baseline. No full-code re-audit was performed. (Refs: current-state §1; evidence-index CLAIM-001; system-map Diagram A; runbook §1)

- `docs/aila/current-state.md` (Refs: current-state §1; evidence-index CLAIM-001; system-map Diagram A; runbook §4)

- `docs/aila/runbook.md` (Refs: current-state §7; evidence-index CLAIM-108; system-map Diagram B; runbook §2)

Default planning assumptions applied:
- `llm.enabled` remains disabled in `live` through Phase 2. (Refs: current-state §5; evidence-index CLAIM-069; system-map Diagram B; runbook §3)
- Roadmap horizon is 12 weeks across 6 two-week sprints. (Refs: current-state §7; evidence-index CLAIM-086; system-map Diagram A; runbook §4)

## Baseline recap
Aila is a Drupal 11 custom module exposed at `/assistant` and `/assistant/api/*`, with a deterministic-first message pipeline (flood checks, validation, safety, out-of-scope, policy, routing, retrieval) and optional LLM enhancement behind config gates. (Refs: current-state §1, §4B, §4E; evidence-index CLAIM-010, CLAIM-033, CLAIM-045, CLAIM-069; system-map Diagram B; runbook §2)

## Prioritization model
Scoring rubric used:
- Impact (1-5)
- Risk reduction (1-5)
- Effort inverse = `(6 - Effort)` where effort scale is `1=small`, `3=medium`, `5=very large`
- Cost sensitivity (1-5)
- Mission alignment (1-5)

Total score formula:
`Total = Impact + RiskReduction + (6 - Effort) + CostSensitivity + MissionAlignment` (Refs: current-state §1; evidence-index CLAIM-001; system-map Diagram A; runbook §4)

Tie-breakers used: lower effort first, then fewer external dependencies. (Refs: current-state §7; evidence-index CLAIM-122; system-map Diagram A; runbook §3)
