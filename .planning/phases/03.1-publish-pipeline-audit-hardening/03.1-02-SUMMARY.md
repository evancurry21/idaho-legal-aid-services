  `gate_vc_pure` aborted before the long-runtime gates ran. **Closed by re-spike**
  in `agent-ae0c8440847532eef`: host composer install (2.8.6) completed in 8s
  with `vendor/bin/phpunit` reporting PHPUnit 11.5.55. The re-spike H1 run then
  drove the full gate path to PASSED.
- The worktree's HEAD is on `worktree-agent-ae0c8440847532eef` (per the
  parallel-executor safety contract), not on `master`. The hook's master-only
  protection blocks (`pre-push-strict.sh:148-171`) are therefore skipped during
  this spike. Wave 2 plan 03's real-push validation should run from a clean
  master checkout to exercise those blocks.
- Local `master` is behind `origin/master`. Git rejected the dry-run as
  non-fast-forward AFTER the hook completed (i.e., AFTER
  `Strict pre-push gate PASSED.` was printed). This rejection is independent of
  the hook's pass/fail and is the source of the dry-run wrapper's exit=1 in
  both runs. The spike's verdict criteria use hook-internal markers
  (PASSED print + zero SIGPIPE) precisely because the wrapper exit code is
  contaminated by this unrelated non-fast-forward state.
